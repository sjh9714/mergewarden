## Show HN (data story, not tool intro)

Numbers below are from the 2026-07-20 run (1,609 scanned, 1,605 complete
analyses); regenerate with `node tools/study/aggregate.mjs` before publishing
if the dataset changes.

Title options (pick one, max ~80 chars):

- Show HN: We scanned 1,609 merged AI-agent PRs – zero declared their intended scope
- Show HN: What 1,609 merged AI-agent PRs actually touched
- Show HN: Scanning 1,609 AI-agent PRs for scope escapes and privilege changes

Body draft:

> AI coding agents (Devin, Copilot coding agent, Codex, Claude Code, Cursor)
> now merge thousands of PRs a day. We scanned 1,609 recently merged
> agent-authored PRs — 299 of them on repositories with 200+ stars, 261 on
> 10k+ — with MergeWarden, a checkout-free policy gate, using its built-in
> default policy only.
>
> What we measured (deterministic evidence, no LLM judgment):
>
> - 7.7% of complete analyses had at least one boundary finding.
> - 4.4% touched agent control-plane files (AGENTS.md, CLAUDE.md,
>   .mcp.json and similar) — the files that steer every future agent PR.
> - Of the 281 PRs that touched GitHub Actions workflows or package
>   manifests, 12.1% escalated workflow permissions and 18.1% introduced
>   unpinned actions.
> - 0 out of 1,609 declared a machine-readable scope contract in the PR body.
> - Finding rates on 10k+ star repositories were roughly 3x lower than on
>   small repositories — established projects have guardrails; the long tail
>   is where agents run unfenced.
>
> Methodology, caveats, and how to reproduce every number:
> [link to docs/study/methodology.md]. Findings are review evidence, not
> vulnerabilities; we publish aggregates only and name no repositories.
>
> MergeWarden (formerly Agent Gate) is MIT-licensed, runs as a checkout-free
> GitHub Action or `npx mergewarden scan owner/repo#123`, never executes PR
> code, and never calls an LLM at runtime. I built it because agent PRs need a
> change-control layer, not another linter. Happy to answer anything about the
> data or the detection rules.
